타입스크립트에서 함수 타입을 정의할 때 Generic 사용하기

Mar 21, 2023 | Aug 28, 2023
2 min read

Generic은 특정 로직을 하나 이상의 타입에 적용할 수 있게 허용하는 도구로써 좀 더 유연하게 타입을 정의할 수 있게 한다.
Generic 타입에 전달하는 number, string과 같은 구체적인 타입을 Type Argument라 한다.

함수 타입을 정의하는 두 가지 방법을 소개하고 두 방법이 무엇이 다른지 알아본다.

  1. type 키워드에서 Generic 사용하기
  2. 함수 타입을 정의할 때 Generic 사용하기

두 방법 모두 type 키워드를 사용해 함수의 파라미터와 리턴 값을 설명하는 함수 타입을 정의하지만, Generic을 사용하는 위치가 다르다.

예제 코드와 함께 보기

예제 코드에서 정의하는 함수 타입(FuncCallApiType)은 함수의 리턴 값을 Generic(<T>)으로 사용한다.

1. type 키워드에서 Generic 사용하기

type 선언문에서 = 기호를 기준으로 왼쪽, 타입 이름 뒤에 Generic <T>를 사용했다.
타입 별칭이 Generic 타입을 포함하는 형식으로. 타입을 사용할 때 Generic의 구체적인 타입을 명시한다.

type FuncCallApiType<T> = (url: string) => Promise<T>;

const doFetch: FuncCallApiType<number> = async (url = '') => {
  const response = await fetch(url);
  if (response.ok) {
    const result = await response.json();
    return result;
  } else 
    return -1;
};

doFetch('/count').then(cnt => console.log(cnt)); // 파라미터 cnt는 number 타입 (Type Inference)

const 키워드로 doFetch 함수를 정의할 때 해당 함수의 타입을 FuncCallApiType으로 지정하면서 Type Argument로 number를 전달했다.

2. 함수 타입을 정의할 때 Generic 사용하기

type 선언문에서 = 기호를 기준으로 오른쪽, 파라미터 리스트 앞에 Generic <T>를 사용했다.
함수가 Generic 타입을 포함하는 형식으로, 함수를 사용할 때 Generic의 구체적인 타입을 명시한다.

type FuncCallApiType = <T>(url: string) => Promise<T>;

const doFetch: FuncCallApiType = async (url = '') => {
  const response = await fetch(url);
  if (response.ok) {
    const result = await response.json();
    return result;
  } else 
    throw Error('Failed');
}

doFetch<string>('/name').then(name => console.log(name)); // 파라미터 name은 string 타입 (Type Inference)

const 키워드로 doFetch 함수를 정의한 후, 함수를 사용할 때 Type Argument로 string을 전달했다.

어떤 방법을 쓸지 결정하기

예제 코드만 본다면 함수가 반환하는 값에 Generic을 활용하고 싶은 것이므로 2번, 함수가 Generic을 포함하도록 구현하는 것이 바람직하다고 생각한다.
1번, 타입 별칭이 Generic을 포함하도록 구현하는 방식은 함수보다 interface 키워드와 조합해 객체에 적용할 때 편리해 보인다. 관련 내용은 Generic Object Types를 참고하면 좋겠다.
